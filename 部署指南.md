# DOTA Auto Eval 部署指南

本指南将帮助您完成 DOTA Auto Eval 系统的部署。

---

## 1. 配置数据库

### 使用 SQLite（默认）

默认情况下，系统使用 SQLite 数据库，无需额外配置。

编辑 `.env` 文件：

```env
DB_TYPE=sqlite
```

### 使用 MySQL

如果需要使用 MySQL 数据库：

编辑 `.env` 文件：

```env
DB_TYPE=mysql
DATABASE_URI=mysql://用户名:密码@主机地址:端口号/数据库名
```

---

## 2. 首次启动并同意 EULA

运行启动脚本：

```bash
cd app_backend
.\start.bat
```

首次启动时，系统会：
1. 自动创建 `eula.txt` 文件（内容为 `False`）
2. 显示用户许可协议（EULA）
3. 提示您同意协议后重启

**按 Ctrl+C 退出程序**

修改 `eula.txt` 文件，将内容改为 `True`：

```txt
True
```

---

## 3. 再次启动并配置 SYSTEM_API_KEY

再次运行启动脚本：

```bash
.\start.bat
```

系统启动成功后，终端会显示类似以下信息：

```
Created system API Key: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

**复制生成的 API Key**，编辑 `.env` 文件：

```env
SYSTEM_API_KEY=您复制的API密钥
```

---

## 4. 配置其他必要参数

编辑 `.env` 文件，配置以下参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| `ADMIN_USERNAME` | 管理员用户名 | `admin` |
| `ADMIN_PASSWORD` | 管理员密码 | `您的密码` |
| `BACKEND_PUBLIC_URL` | 后端服务器公开地址 | `http://your-server-ip:5000` |

**示例配置：**

```env
ADMIN_USERNAME=admin
ADMIN_PASSWORD=YourSecurePassword123
BACKEND_PUBLIC_URL=http://192.168.1.100:5000
```

---

## 5. 启动服务器

完成以上所有配置后，运行：

```bash
.\start.bat
```

系统将正常启动，您可以访问前端页面开始使用。

---

## 配置清单

完成部署后，您的 `.env` 文件应包含以下内容：

```env
# 数据库配置
DB_TYPE=sqlite  # 或 mysql
DATABASE_URI=   # MySQL 时需要填写

# 管理员账户
ADMIN_USERNAME=admin
ADMIN_PASSWORD=您的密码

# 后端地址
BACKEND_PUBLIC_URL=http://您的服务器IP:5000

# 系统API密钥
SYSTEM_API_KEY=您生成的API密钥
```

## 常见问题

### Q: 如何修改端口号？

A: 编辑 `.env` 文件，添加或修改 `SERVER_PORT` 参数：

```env
SERVER_PORT=8080
```

### Q: 如何开启调试模式？

A: 编辑 `start.bat`，将 `debug=False` 改为 `debug=True`：

```bash
python app.py --debug
```

### Q: 如何查看数据库内容？

A: 使用 SQLite 命令行工具：

```bash
sqlite3 app_backend/instance/dota_auto_eval.db
```
